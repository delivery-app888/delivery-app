# 配達ログアプリ — 事業計画・ロードマップ

*最終更新: 2026年3月*

---

## 1. プロダクト概要

### コンセプト
フードデリバリー配達員の稼働データを記録・可視化し、「自分の稼ぎ方を理解できる」ツール。
単なる記録アプリではなく、データから行動提案まで踏み込むことで差別化する。

### ターゲット
- 年齢: 40代後半〜60代
- 収入: 年収400〜700万、老後資金に不安
- 性格: 堅実・真面目・コツコツ型
- ITリテラシー: 普通（スマホ・LINE・YouTube・検索は使える）
- 目標: 月5〜20万円の副収入、自分のペースで働きたい

### 最終ゴール
フードデリバリーを「ただのバイト」ではなく「自立した収入源」に変える

## 2. 実装済み機能一覧（Claude.aiアーティファクト版）

### 基本記録機能（全て無料）
- 天候ゲート（天候未選択でオンライン不可、5択ポップアップ）
- オンライン/オフライン（複数セッション対応）
- 休憩開始/終了
- 受注/配達完了/報酬入力
- シングル/ダブル/トリプル選択
- 調理待ちキャンセル記録
- 地蔵ボタン（初回タップ時に説明ポップアップ表示）
- 日次インセンティブ入力
- 配達履歴タップ編集（報酬・インセンティブ・会社・タイプ・評価の変更・削除）
- 配達評価（🟡良い / ⚪普通 / 🔵悪い、自動判定＋手動上書き）

### UI/UX
- ダークモード/ライトモード切り替え
- 文字サイズ（標準/大きめ）トグル — 老眼対策
- ハンバーガーメニュー（☰）→ アコーディオン式分析メニュー
- 初回チュートリアル（5ステップ、スキップ可能）
- アクショントースト（ボタン押下時の確認フィードバック）
- おつかれさまカード（オフライン時に成績サマリー表示、自己ベスト更新通知）
- 連続稼働日数（ストリーク）ヘッダー表示
- 今日の目安トースト（オンライン開始時に日割り目標を表示）
- AutoFitText（金額の桁数増加時にフォント自動縮小）

### 分析機能
- 7つのサブ画面に分離（アコーディオンメニューから選択）
  - 📋 デイリーレポート（無料）— 棒グラフ＋円グラフ付き
  - ⏰ 時間帯分析（プレミアム）— 8時間帯の売上＋単価棒グラフ
  - 📅 曜日分析（プレミアム）— 曜日別平均売上棒グラフ
  - 🏢 会社別分析（プレミアム）— 円グラフ＋平均単価比較
  - 🌤️ 天候分析（プレミアム）— 天候別時給横棒グラフ
  - 📈 推移（プレミアム）— 週別＋月別売上の折れ線グラフ
  - 💰 平均単価（プレミアム）— 4期間切り替え（今日/今週/今月/全期間）
- 稼ぎ方ポイント（データから自動生成されるアドバイス）
- プレミアム画面のぼかし表示（blur 3px）→ チラ見せ戦略

### 稼働情報の時間内訳
- 稼働時間（全体）
- 実配達時間（受注→配達完了の合計）
- 無職時間（稼働 − 実配達 − 休憩、うち地蔵時間を内訳表示）
- 休憩時間

### 設定
- ダークモード ON/OFF
- 文字サイズ（大きめ）ON/OFF
- 配達報酬にインセンティブを含める ON/OFF
- 月間目標にインセンティブを含める ON/OFF
- デモデータ生成（6ヶ月分、開発用）

### データ構造（deliveryオブジェクト）
```
{
  orderTime, completeTime, company, reward, incentive,
  orderType, cancelled,
  rating,                    // "good"|"normal"|"bad"|null（自動判定＋手動上書き）
  startLat, startLng,        // GPS受注地点（PWA化で実装）
  endLat, endLng             // GPS配達完了地点（PWA化で実装）
}
```

### ストレージ
- キー: `log:YYYY-MM-DD`（日別）、`all-logs-index`、`monthly-goal`、`app-settings`
- Claude環境: `window.storage`（キーバリュー）
- PWA移行後: IndexedDB

## 3. マネタイズ戦略

### 収益モデル: フリーミアム

#### 無料版（永久無料）
- 全ての記録機能（制限なし）
- デイリーレポート（当日の棒グラフ＋円グラフ）
- 当日のMAPピン表示（PWA化後）
- 月間目標ゲージ
- 設定（テーマ・文字サイズ・インセンティブ設定）
- 稼ぎ方ポイント（1つだけ無料表示）

#### プレミアム版（月額580円 / 年額4,800円）
- 時間帯・曜日・天候・会社別の詳細分析
- 週次/月次の売上推移グラフ
- 平均単価の期間切り替え
- 全期間の稼ぎ方ポイント（全項目表示）
- 過去全期間のMAPピン表示＋フィルター機能
- 配達履歴の検索・絞り込み
- CSVエクスポート
- クラウドバックアップ（機種変更対応）

#### チラ見せ戦略
- プレミアム画面はコンテンツをblur(3px)でぼかし表示
- デイリーレポートの各セクション末尾に「過去30日の傾向と比較 →」等の導線
- 完全ロックではなく「もう少しで見える」感覚で好奇心を刺激

#### 無料→有料の転換設計
- 1〜7日目: 全機能開放（体験期間、UI表示なし）
- 8日目以降: 分析画面の一部がロック開始
- 2週間後: 「データが溜まりました。詳しい分析を見てみましょう」と誘導
- 1ヶ月後: 最大の課金タイミング（先月との比較が見たくなる）

### 追加収益源
- 有料コンテンツ（エリア攻略法、時間帯別立ち回り等の記事/動画、買い切り300〜500円）
- アフィリエイト（保温バッグ・スマホホルダー等の配達員グッズ）
- 配達員登録紹介（各社の紹介コード）

### 売上シミュレーション（控えめ見積もり）

| 期間 | 累積ユーザー | 課金者（5%） | 月間売上 |
|------|------------|-------------|---------|
| 3ヶ月 | 1,500 | 75 | 43,500円 |
| 6ヶ月 | 3,000 | 150 | 87,000円 |
| 1年 | 6,000 | 300 | 174,000円 |
| 2年 | 15,000 | 750 | 435,000円 |

＋アフィリエイト・コンテンツで1年後に月20〜30万、2年後に月50万以上を目指す

## 4. 集客・マーケティング戦略

### 最重要チャネル: YouTube
- 配達しながらアプリを使う動画を週3本投稿
- 「アプリの宣伝」ではなく「配達のリアル」を見せる
- アプリは「証拠」として自然に登場させる
- 概要欄にアプリリンク

### X（Twitter）配達員コミュニティ
- #ウーバーイーツ #出前館 で毎日稼働データのスクショを投稿
- 自然にアプリの存在が広まる

### SEOブログ
- 「Uber Eats 時給」「配達員 副業 始め方」等のキーワードで記事作成
- 記事内でアプリを紹介

### 口コミ（最強の武器）
- 無料版が十分便利 → ユーザーが勝手に広める
- 配達員は待機中に情報交換する文化がある

### 初期スケジュール
- 1ヶ月目: 自分で使ってデータ蓄積、YouTube週3本、X毎日投稿 → 目標100DL
- 2〜3ヶ月目: コミュニティ参加、SEOブログ5〜10記事 → 目標500DL
- 4〜6ヶ月目: プレミアムリリース → 目標課金者50人（月29,000円）

### 広告費: ゼロでスタート
ペルソナが「怪しいビジネス嫌い」のため、広告よりコンテンツと口コミで信頼構築

## 5. 技術ロードマップ

### フェーズ1: Claude.aiアーティファクト（現在）✅
- 基本記録機能の全実装
- 分析画面（グラフ付き7サブ画面）
- フリーミアムUI（ぼかし表示、チラ見せ導線）

- 配達評価（自動判定＋手動上書き）
- GPS・MAP用データ構造の準備（startLat/Lng, endLat/Lng）

### フェーズ2: PWA化（最短1〜2週間）
- index.html化 + Service Worker（オフライン対応）
- `window.storage` → IndexedDB移行
- manifest.json（アプリ名、アイコン、テーマカラー）
- GitHub Pages or Cloudflare Pages にデプロイ
- GPS取得実装（navigator.geolocation）
- 気象API連携（Open-Meteo、無料・APIキー不要）
- MAP表示（Leaflet + OpenStreetMap、完全無料）
  - 配達評価に応じたピンの色分け（🟡良い / ⚪普通 / 🔵悪い / 🔴キャンセル）
  - 無料版: 当日のピンのみ表示
  - プレミアム: 全期間＋時間帯/曜日/会社/評価フィルター
- 課金機能（Stripe Checkout or RevenueCat）

### フェーズ3: ストアアプリ化（成長後）
- React Native or Flutter でネイティブ化
- App Store / Google Play 公開
- App内課金（Apple/Google）
- プッシュ通知（稼働リマインダー、週次まとめ）
- バックグラウンドGPS
- クラウドバックアップ（プレミアム機能）

### フェーズ4: 機能拡充
- 店舗管理（GPS座標ベースの自動特定 + Nominatim API）
- エリアヒートマップ（平均単価の地域分布）

- ユーザー間のデータ共有（匿名エリア分析等）

## 6. データ量・パフォーマンス

### 10年分のデータ保持が可能
- 50件/日 × 365日 × 10年 = 182,500件 ≒ 約36MB
- IndexedDB: 端末容量の50%まで利用可能 → 余裕
- SQLite（ネイティブアプリ化時）: 制限なし
- 10万件の集計処理: JavaScript上で1秒以内

## 7. プライバシー・セキュリティ

### 設計方針: 全データがユーザーの端末内に閉じる

| 技術 | 外部送信される情報 | リスク |
|------|-------------------|--------|
| ブラウザGPS | なし | なし |
| Leaflet | なし | なし |
| OpenStreetMap | IPアドレス＋閲覧エリア | 極めて低い |
| Open-Meteo | 緯度経度 | 低い（GDPR準拠） |
| IndexedDB | なし | なし |

### 必要な法的対応（PWA/ストア公開時）
- プライバシーポリシーの作成と公開
- 特定商取引法に基づく表記（有料機能提供時）
- 位置情報の利用目的の明示

### OpenStreetMapの商用利用
- 小規模（〜数百ユーザー）: OSMタイル直接利用で問題なし
- 成長後: MapTiler（月10万リクエスト無料）またはStadia Mapsに切り替え
- 切り替え時にピンデータ・位置情報は一切影響なし（地図タイルURLの変更のみ）
